# STT API 벤치마크 결과표

> 정량 지표(1~3장)는 `uv run python scripts/build_report.py` 및 DB 집계로 채웠다.
> 기능 검증(4장)·종합 평가(5~6장)는 실측 가능한 항목을 채우고, 수동 측정이 필요한 항목은 명시했다.
> 판단 기준·가중치의 근거는 [설계서](./STT_API_선정_프로젝트_설계서.md) 10~11장 참고.

- 실험일: 2026-07-07
- 데이터: AI Hub 채용면접 인터뷰 데이터(ICT), 답변 wav **8개** (여/남 × 신입/경력 각 2)
- 실행 환경: Windows 11 / Python 3.12 / CPU (SenseVoice 로컬 추론 포함)
- 대상 4종 전부 실측 완료 (Clova, Vito, Whisper, SenseVoice-Small)

## 1. 정확도 (CER 1차 지표)

| API | n | CER | WER | 문장정확도 | Latency(s) | RTF | 추임새/건 | 더듬음/건 | SPS |
|---|--:|--:|--:|--:|--:|--:|--:|--:|--:|
| vito | 8 | **0.023** | 0.075 | 0.250 | 2.1 | 0.03 | 6.0 | 0.2 | 5.15 |
| clova | 8 | 0.040 | 0.115 | 0.151 | 3.7 | 0.05 | 2.0 | 0.2 | 7.36 |
| sensevoice | 8 | 0.066 | 0.451 | 0.109 | 7.7 | 0.10 | 2.2 | 0.2 | 11.90 |
| whisper | 8 | 0.075 | 0.199 | 0.060 | 5.2 | 0.06 | 0.2 | 0.2 | 3.87 |

관찰:
- **CER 순위: Vito(2.3%) < Clova(4.0%) < SenseVoice(6.6%) < Whisper(7.5%)**. 국산 API 2종이 상위.
- WER은 CER의 3~7배 → 한국어 띄어쓰기 변동 때문. **CER을 1차 지표로 삼은 설계 판단이 실측으로 확인됨.**
- **Whisper의 추임새 0.2건/파일** — 간투어를 거의 다 제거함(4.1 참고). CER도 최하위.
- SenseVoice의 SPS 11.9는 비정상적으로 높음 → timestamp가 단어 span이 아닌 onset이라 발화시간 과소 측정(4.2 참고).

## 2. 그룹별 편차 (성별 × 경력)

| API | 여성 CER | 남성 CER | 신입 CER | 경력 CER | 편차 |
|---|--:|--:|--:|--:|--:|
| vito | 0.020 | 0.025 | 0.019 | 0.026 | **0.007** |
| clova | 0.041 | 0.039 | 0.049 | 0.031 | 0.018 |
| sensevoice | 0.064 | 0.068 | 0.075 | 0.057 | 0.018 |
| whisper | 0.072 | 0.078 | 0.090 | 0.059 | 0.031 |

관찰:
- 네 엔진 모두 성별 편차는 미미(≤0.006). **면접 지원자 성별에 따른 편향은 없음.**
- 모든 엔진에서 경력 그룹 CER이 신입보다 낮음 → 경력자 발화가 더 또렷/정연한 경향.
- **Vito는 편차 0.007로 가장 안정적**, Whisper는 0.031로 신입/경력 간 격차가 가장 큼.

## 3. 응답속도

합격선(설계서 5.4): 배치 RTF < 0.5, latency < 오디오길이의 30%(≈25s)

| API | 평균 Latency(s) | 평균 RTF | 최대 Latency(s) | 합격 여부 |
|---|--:|--:|--:|:-:|
| vito | 2.1 | 0.03 | 2.7 | ✅ |
| clova | 3.7 | 0.05 | 3.9 | ✅ |
| whisper | 5.2 | 0.06 | 6.5 | ✅ |
| sensevoice | 7.7 | 0.10 | 8.5 | ✅ |

- 네 엔진 모두 합격선을 크게 통과. 배치 면접 분석에는 응답속도가 변별 요인이 아님.
- SenseVoice는 로컬 CPU 추론임에도 RTF 0.10 — "개빠르다"는 평판 확인.

## 4. 기능 검증 (면접 특화)

### 4.1 Verbatim / 간투어 보존 ★ 핵심 지표

GT(사람 verbatim 전사)의 간투어("아","어","네","그","음","저","뭐","이제","약간","저기") 대비
각 엔진 전사에 남은 간투어 비율. GT 8개 파일 총 79개 기준.

| API | 전사 간투어 수 | GT 대비 보존율 | 판정 |
|---|--:|--:|:-:|
| **vito** | 82 | **104%** | ◎ 완전 보존 (오히려 더 탐지) |
| clova | 23 | 29% | △ 대부분 필터링 |
| sensevoice | 21 | 27% | △ 대부분 필터링 |
| whisper | 10 | **13%** | ✗ 거의 전부 제거 |

- **이 항목이 엔진 간 가장 큰 차이를 만든다.** Vito만 간투어를 온전히 전사 → 더듬음·추임새 평가가 성립.
- Clova/SenseVoice는 ~70%, **Whisper는 ~87%를 지워버려**, 후처리로 추임새를 세는 것 자체가 불가능해짐.
- Whisper의 낮은 보존율은 사전 예측("Whisper는 간투어/반복 제거 경향")이 **실측으로 확인된 결과**.
- 보존율 104%는 Vito가 GT보다 더 많이 탐지했다는 의미(과탐 소지)이나, 면접 평가에서는 "지우지 않음"이 훨씬 중요.

### 4.2 Word Timestamp

| API | 제공 여부 | 단위 | 수동 라벨 대비 평균 오차(ms) | 비고 |
|---|:-:|---|--:|---|
| vito | ✅ | word span | 수동 측정 필요 | 시작+지속시간 제공 |
| clova | ✅ | word span | 수동 측정 필요 | segment 내 word 정렬 |
| whisper | ✅ | word span | 수동 측정 필요 | verbose_json word 단위 |
| sensevoice | ⚠️ | CTC onset (60ms) | 수동 측정 필요 | span 아님 → **발화시간 과소, SPS 왜곡** |

- SenseVoice의 timestamp는 토큰 발화 "시작점"만 신뢰 가능. 말속도·침묵 계산의 정밀도가 상용 API보다 낮음.
- ms 단위 정확도는 샘플 5개 Audacity 수동 라벨링 후 갱신 예정(설계서 10.2).

### 4.3 Confidence

| API | 제공 단위 | 활용 가능성 |
|---|---|---|
| vito | word/segment | 저신뢰 구간 표시, 재검토 플래그 |
| clova | segment | segment 단위 신뢰도 필터 |
| whisper | ❌ word 미제공 (segment avg_logprob만) | 제한적 |
| sensevoice | ❌ 미제공 | — |

### 4.4 침묵 측정 일치도 (STT gap vs webrtcvad)

- `app/analysis/silence.py`에 STT gap 방식과 webrtcvad 방식이 모두 구현됨.
- 두 방식의 IoU 비교는 추가 분석으로 진행 예정(미측정). SenseVoice는 4.2의 timestamp 한계로 STT gap 방식 신뢰도가 낮을 것으로 예상.

## 6. 최종 결론

> 우리는 **Vito(리턴제로)**를 선택한다. 근거:
> ① CER **2.3%로 1위** (Clova 4.0%, SenseVoice 6.6%, Whisper 7.5%)
> ② **간투어 보존율 104%** — 4개 중 유일하게 verbatim 전사가 가능해 더듬음·추임새 평가가 성립하는 엔진
>    (Clova 29%, SenseVoice 27%, Whisper 13%)
> ③ word span timestamp + word 단위 confidence 제공, 그룹 편차 최소(0.007)
> ④ 응답속도 RTF 0.03으로 합격선 여유 통과
> ⑤ 미지원 지표(억양, 발음막힘)는 어떤 엔진을 선택해도 별도 음향분석이 필요하므로 감점 사유가 아님.

**핵심 논리**: 정확도(CER)만 보면 Vito·Clova 격차는 1.7%p에 불과하지만, **면접 평가의 성립 조건인 간투어 보존에서 Vito(104%)와 나머지 전부(≤29%)의 차이가 결정적**이다. Clova·Whisper·SenseVoice를 택하면 간투어의 70~87%가 사라져 추임새·더듬음 지표를 아예 계산할 수 없고, "AI 면접 평가"라는 목적 자체가 무너진다.

**주목할 점 — Whisper의 탈락**: Whisper는 비용(시간당 ≈500원)·구현 난이도에서 최상위지만, 간투어 보존율 13%로 면접 평가용으로는 부적합. "일반 STT 벤치마크에서 좋은 엔진(간투어를 깔끔히 제거)이 면접 평가에서는 오히려 탈락한다"는 이 프로젝트의 핵심 논지를 실측으로 보여주는 사례.

**확정 후 남은 정밀 검증(선정 결론은 불변)**:
1. word timestamp ms 오차 수동 측정 — Vito vs Clova의 timestamp 정확도 우열 확정용(선정은 이미 Vito).
2. 침묵 측정 STT gap vs VAD 일치도(4.4).
