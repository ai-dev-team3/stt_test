# 포팅 메뉴얼 (Porting Manual)

> STT API 선정 벤치마크 프로젝트를 새 환경에 설치·실행하는 절차.
> 대상 OS: Windows / macOS / Linux (개발 환경은 Windows 11 기준)

---

## 1. 요구사항

| 항목 | 버전 | 비고 |
|---|---|---|
| [uv](https://docs.astral.sh/uv/) | 0.11+ | Python 버전·의존성 모두 uv가 관리 |
| Python | 3.12 | **직접 설치 불필요** — uv가 자동 다운로드 |
| Git | - | |

uv 설치 (없는 경우):

```powershell
# Windows
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
```

## 2. 설치

```bash
git clone <repo-url> stt_test
cd stt_test
uv sync          # .venv 생성 + uv.lock 기준 전체 의존성 설치 (Python 3.12 자동 확보)
```

설치 확인:

```bash
uv run python -c "import librosa, parselmouth, webrtcvad, jiwer, fastapi, funasr; print('OK')"
```

## 3. 데이터 배치

데이터셋(AI Hub 채용면접 인터뷰 데이터)은 라이선스상 저장소에 포함되지 않는다 (`.gitignore`).
AI Hub에서 다운로드 후 아래 구조로 배치:

```
data/
├── origin/                          # wav (16kHz/16bit/mono)
│   ├── Female/{Experienced,New}/    #   ckmk_q_* = 질문, ckmk_a_* = 답변(평가 대상)
│   └── Male/{Experienced,New}/
└── label/                           # 정답 라벨 JSON (ckmk_d_*)
    ├── Female/{Experienced,New}/
    └── Male/{Experienced,New}/
```

- 라벨 JSON의 `dataSet.answer.raw.text`가 Ground Truth로 사용된다
- 라벨 파일명 `ckmk_d_..._{id}.json` ↔ 답변 wav `ckmk_a_..._{id}.wav` 로 자동 매칭된다

## 4. API 키 설정

```bash
cp .env.example .env    # 후 발급받은 키 입력
```

| 엔진 | 발급 위치 | 필요 값 |
|---|---|---|
| Clova Speech | [NCP 콘솔](https://console.ncloud.com) → AI Services → CLOVA Speech → 도메인 생성 | `CLOVA_INVOKE_URL`, `CLOVA_SECRET_KEY` |
| Vito (리턴제로) | [developers.rtzr.ai](https://developers.rtzr.ai) → 애플리케이션 등록 | `VITO_CLIENT_ID`, `VITO_CLIENT_SECRET` |
| Whisper API | [platform.openai.com](https://platform.openai.com/api-keys) → API 키 발급 | `OPENAI_API_KEY` |
| SenseVoice-Small | **키 불필요** (로컬 모델) — 최초 실행 시 모델 ~1GB 자동 다운로드 | 없음 |

- **키가 없는 API는 자동으로 건너뛴다** — 키가 하나도 없어도 SenseVoice·데이터 적재·분석 모듈은 동작
- `.env`는 gitignore 대상. 절대 커밋하지 말 것

## 5. 실행

### 5.1 벤치마크 (메인)

```bash
uv run python scripts/run_benchmark.py           # 키가 설정된 모든 API 실행
uv run python scripts/run_benchmark.py clova     # 특정 API만
```

- 결과는 프로젝트 루트 `stt_test.db`(SQLite)에 저장
- **이미 결과가 있는 (음성 × API) 조합은 API를 재호출하지 않는다** (비용 보호) — 중간 실패 시 그냥 재실행하면 이어서 진행
- 강제로 다시 돌리려면 `stt_test.db` 삭제 후 실행

### 5.2 결과표 생성

```bash
uv run python scripts/build_report.py                          # 콘솔 출력
uv run python scripts/build_report.py docs/결과표_생성본.md     # 파일로 저장 (UTF-8)
```

### 5.3 FastAPI 서버 (선택)

```bash
uv run uvicorn app.main:app --reload
```

| 엔드포인트 | 설명 |
|---|---|
| `GET /health` | 헬스체크 |
| `POST /evaluate/{audio_id}?apis=clova,vito` | 음성 1개 평가 (apis 생략 시 설정된 전체) |
| `GET /report` | API별 평균 지표 JSON |
| `GET /docs` | Swagger UI |

## 6. 결과 저장 구조

`stt_test.db` (SQLite, gitignore 대상):

| 테이블 | 내용 |
|---|---|
| `audio_files` | 음성 메타 + 정답 텍스트 (라벨에서 적재) |
| `stt_results` | 파일×API별 전사 결과, latency, RTF, **원본 응답 JSON**(`raw_response`) |
| `stt_words` | 단어별 timestamp / confidence |
| `metrics` | 파일×API별 CER/WER/문장정확도/말속도/더듬음/추임새/침묵/발화비율/억양 |

원본 응답을 보존하므로, 지표 정의를 바꿔도 API 재호출 없이 재계산 가능하다.

## 7. 트러블슈팅

| 증상 | 원인 / 해결 |
|---|---|
| 콘솔에서 한글이 `����`로 깨짐 | Windows 콘솔 cp949 문제. 데이터는 정상. `$env:PYTHONUTF8=1` 설정 또는 무시 |
| SenseVoice 최초 실행이 오래 걸림 | 모델 ~1GB 다운로드 중 (1회만). 이후에는 로컬 캐시 사용 |
| Whisper 413/400 | 파일 25MB 제한 — 본 데이터셋(2~3MB)은 해당 없음 |
| Vito 401 | 토큰은 요청마다 갱신하므로 `VITO_CLIENT_ID/SECRET` 값 자체를 확인 |
| `webrtcvad` 설치 실패 | 이 프로젝트는 빌드가 필요 없는 `webrtcvad-wheels`를 사용 — `uv sync`로 설치했는지 확인 |
| 벤치마크 재실행 시 API가 호출되지 않음 | 정상 (결과 재사용). 재호출하려면 `stt_test.db` 삭제 |

## 8. 프로젝트 구조

```
stt_test/
├── app/
│   ├── main.py              # FastAPI 엔트리포인트
│   ├── config.py            # .env 로딩 (pydantic-settings)
│   ├── pipeline.py          # 음성×API 실행 → DB 저장 (결과 재사용 포함)
│   ├── db.py / models.py    # SQLAlchemy (SQLite)
│   ├── adapters/            # STT 어댑터 4종 → 공통 스키마 정규화
│   ├── analysis/            # 정확도·말속도·더듬음·추임새·침묵·억양
│   └── routers/             # /evaluate, /report
├── scripts/
│   ├── run_benchmark.py     # 일괄 실행
│   └── build_report.py      # 비교표 생성
├── data/                    # 데이터셋 (git 미포함)
└── docs/                    # 설계서, 포팅 메뉴얼, 결과표
```
